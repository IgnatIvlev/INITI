Для реализации данной задачи можно использовать объект типа Map, который будет в качестве ключа будет
использовать уникальный ID в качестве значения будет объект Row. Поэтому я считаю, что нужно 5 классов(без
учета вспомогательных классов такие как данные пользователей и т.д.):

Row - класс отвечающий за работу со строкой.

Table - за работу с таблицей.

Client - клиентский класс, отображает данные таблице у клиента, формирует запросы на получение данных
в зависимости от скролинга.

Server - Хранит уникальную таблицу. Осуществляет обработку запроса клиента и передачу необходимых данных
из класса Table.
    Класс Sorter(внутеренний класс Server) - вспомогательный класс, содержащий методы сортировки данных в таблице.
__________________________________________
Реализовывать необходимо используя многопоточность.
В процессе разарботки могут понадобится дополнительные классы.
